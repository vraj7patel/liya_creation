# Liya Creation - Premium Ethnic Wear E-commerce Platform

Liya Creation is a comprehensive full-stack e-commerce application designed for premium ethnic wear, built as a BCA project showcasing modern web development practices. The platform features a rich product catalog, shopping cart, wishlist, order management, and dedicated admin dashboard. It targets users interested in high-quality ethnic attire such as lehengas, sarees, gowns, and kurtis, with support for multiple sizes, image galleries, and stock management. The application emphasizes responsive design, secure authentication, and seamless user experience across browsing, purchasing, and account management.

The technical architecture follows the MEAN stack: MongoDB for data persistence, Express.js and Node.js for the backend API server, Angular 17 for the frontend single-page application (SPA), and comprehensive integration via RESTful APIs. The backend runs on localhost:3000 with Nodemon for development hot-reloading, while the Angular frontend operates on localhost:4200, proxying API calls to avoid CORS issues. Key dependencies include Mongoose for schema-based MongoDB interactions, Passport.js for authentication strategies, Multer for file uploads, and Nodemailer for email functionalities.

At the core of the backend are well-structured Mongoose models including Product (with fields like name, description, price, category, images, sizes, stock, isFeatured), User, Order, Cart, Category, and Wishlist. Controllers handle CRUD operations for these entities, with routes prefixed under /api/ (e.g., /api/products, /api/cart, /api/orders). Middleware ensures authentication (isAuthenticated, isAdmin) and handles multipart image uploads for products, storing them in /uploads/products/. Security features include bcryptjs for password hashing, session management, and role-based access control.

The Angular frontend is organized with lazy-loaded modules, feature components, and shared UI elements like premium-product-card, shop-by-category, header, and footer. Core services (auth, product, cart, wishlist, order) manage state via RxJS Observables and BehaviorSubjects. Guards (auth.guard.ts, admin.guard.ts) protect routes, while an auth interceptor automatically handles tokens/sessions for API requests. User-facing pages include product lists/details, cart, wishlist, checkout, user dashboard/orders, and static informational pages (FAQ, shipping, returns, size guide, contact).

Authentication is robust, supporting local login/register, Google OAuth via Passport.js, and Clerk integration. Admin functionalities enable product/category CRUD with image uploads, order oversight, and user management. E-commerce flows cover product browsing with filters, quick-view modals, cart persistence, order placement, tracking, and history. The platform supports featured products, category-based shopping, and responsive design for mobile/desktop.

Development setup is straightforward: install dependencies in both frontend/backend directories, start MongoDB, run backend with 'npm run dev', frontend with 'ng serve', and optionally seed the database. Common issues like API errors or MongoDB connections are documented, with fixes for Multer validation and CORS. This project demonstrates enterprise-level patterns including reactive forms, HTTP interceptors, lazy loading, and secure file handling, making it an excellent reference for full-stack e-commerce development.
