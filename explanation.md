# Why `npm start` instead of `ng serve`? Project Explanation

## Project Overview
This is a **full-stack e-commerce application** called **Liya Creation** with:
- **Frontend**: Angular 17 application in `liya-frontend/`
- **Backend**: Node.js/Express API in `liya-backend/`

## Why `npm start` runs the Angular project

### 1. **Standard Angular CLI Scripts**
The `liya-frontend/package.json` defines these standard Angular scripts:

```json
{
  "scripts": {
    "ng": "ng",
    "start": "ng serve",
    "build": "ng build",
    "watch": "ng build --watch --configuration development",
    "test": "ng test"
  }
}
```

- **`npm start`** → executes `ng serve`
- **`ng serve`** is Angular CLI's **development server** that:
  - Compiles TypeScript to JavaScript
  - Serves files at `http://localhost:4200` (default)
  - Enables hot reload (auto-refresh on changes)
  - Provides live error reporting

### 2. **Why NOT direct `ng serve`?**
```
✅ npm start (RECOMMENDED)
   - Uses package.json scripts
   - Consistent across teams/environments
   - Can be customized (ports, proxies, etc.)
   - Cross-platform compatible

❌ Direct ng serve
   - Requires Angular CLI globally installed
   - Version mismatches possible
   - No custom configurations from package.json
```

### 3. **How to run the project**

#### Frontend (Angular)
```bash
cd liya-frontend
npm install          # Install dependencies
npm start            # Runs: ng serve → http://localhost:4200
```

#### Backend (Node.js/Express)
```bash
cd liya-backend
npm install          # Install dependencies
npm start            # Runs backend server (check server.js for exact command)
```

### 4. **Proxy Configuration**
Angular uses `proxy.conf.json` to forward API calls to backend:
```
Frontend (4200) → proxies → Backend (typically 5000)
```
